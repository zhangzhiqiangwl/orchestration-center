# 编排中心开发指南

## 特性介绍

本特性允许用户为不同操作定义自定义实现，同时为常见操作提供默认实现。通过统一的抽象基类和处理器注册机制，开发者可以灵活扩展系统功能，无需修改核心代码。

**核心能力：**
- **抽象基类**：为所有处理器定义统一接口
- **默认实现**：为常见操作提供内置处理器
- **自定义扩展**：支持用户注册自定义处理器

## 约束与限制

- **功能限制**：
  - 自定义处理器必须继承`BaseHandle`并实现`handle`方法
  - 同一接口类型只能注册一个处理器，后注册的会覆盖先注册的
  - 不支持运行时动态卸载已注册的处理器
- **操作限制**：
  - 注册操作应在应用启动时完成，避免在业务处理过程中注册

## 环境准备

### 环境要求

- Python 3.10+

### 搭建环境

该模块设计为Python项目的组成部分，直接包含在项目结构中即可使用。确保以下文件在项目目录中：
- `default_handle.py` - 包含基类、注册表和默认处理器
- `custom_handle.py` - 自定义处理器扩展模块（如需要）

### 检验环境是否搭建成功

```python

# 验证默认处理器是否可用

print(f"处理器加载成功: {type(handler).__name__}")
```
## 系统架构

系统采用注册表模式，核心组件包括：

| 组件 | 职责 |
|------|------|
| BaseHandle | 定义所有处理器的统一抽象接口 |
| HandlerRegistry | 管理处理器注册和获取，提供默认实现兜底 |
| InterfaceType | 定义支持的接口类型枚举 |
| 默认处理器 | 为常见操作提供内置实现 |
| 自定义处理器 | 用户扩展实现，覆盖默认行为 |

```
┌─────────────────┐
│   业务调用方     │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ HandlerRegistry │◄──── 注册自定义处理器
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│   BaseHandle    │
│   (抽象基类)     │
└────────┬────────┘
     ┌───┴───┬───────────────┐
     ▼       ▼               ▼
┌───────┐ ┌───────┐  ┌───────────┐
│ 默认  │  │ 默认 │   │  自定义   │
│处理器1│  │处理器2│  │  处理器3  │
└───────┘ └───────┘  └───────────┘
```
## 接口说明

| 类/接口名 | 描述 |
|-----------|------|
| BaseHandle | 所有处理器必须继承的抽象基类 |
| BaseHandle.handle() | 需要子类实现的抽象异步方法 |
| HandlerRegistry | 处理器注册表，管理注册和获取 |
| HandlerRegistry.register() | 注册处理器类到指定接口类型 |
| HandlerRegistry.get_handler() | 获取指定接口类型的处理器实例 |
| InterfaceType | 接口类型枚举（如QUERY、SAVE等） |
| SavePsopHandler | 保存psop的默认处理器 |
| GetAllPsopsHandler | 查询所有psop的默认处理器 |
| GetPsopHandler | 通过id查询psop的默认处理器 |
| DeletePsopHandler | 删除psop的默认处理器 |

## 开发步骤

### 步骤1：引入所需模块
```python
from default_handle import BaseHandle, HandlerRegistry, InterfaceType
```

### 步骤2：创建自定义处理器
创建继承自BaseHandle的自定义类，并实现handle方法：
```python
class MyCustomHandle(BaseHandle):
    """自定义处理器示例"""
    
    def handle(self, *args, **kwargs):
        """
        自定义处理逻辑
        
        Args:
            *args: 位置参数
            **kwargs: 关键字参数
            
        Returns:
            处理结果
        """
        # 自定义实现
        # 建议：在这里添加具体的业务逻辑
        # 注意：确保方法是异步的
        result = "自定义结果"
        return result
```

### 步骤3：注册自定义处理器
在应用启动时注册自定义处理器：
```python
# 注册自定义处理器
# 注意：注册操作应在业务处理开始前完成
HandlerRegistry.register(InterfaceType.QUERY, MyCustomHandle)
```
警告：后注册的处理器会覆盖同类型的先前处理器。请确保注册顺序符合预期。
### 步骤4：使用处理器
在业务代码中获取并使用处理器：
```python
# 获取处理器实例
# HandlerRegistry会自动返回已注册的自定义处理器或默认实现
handle = HandlerRegistry.get_handler(InterfaceType.QUERY)

# 使用处理器执行业务
result = handle.handle(...)
```
### 步骤5：完整示例
以下是一个完整的自定义处理器实现示例：
```python
from default_handle import BaseHandle, HandlerRegistry, InterfaceType

class SaveCustomHandle(BaseHandle):
    """自定义查询处理器"""
    
    def handle(self, query_params=None):
        """
        自定义查询逻辑
        
        Args:
            query_params: 查询参数字典
            
        Returns:
            查询结果列表
        """
        # 示例：实现自定义查询逻辑
        if not query_params:
            return []
        
        # 自定义查询实现
        results = []
        # ... 业务逻辑
        return results

# 注册
HandlerRegistry.register(InterfaceType.SAVE_PSOP, SaveCustomHandle)

# 使用
handler = HandlerRegistry.get_handler(InterfaceType.SAVE_PSOP)
data = handler.handle({"id": 1})
print(data)
```
## 默认处理器说明

若未注册自定义处理器，系统将使用以下默认实现：

| 处理器 | 对应接口类型 | 功能描述 |
|--------|-------------|----------|
| SavePsopHandler | SAVE | 保存psop |
| GetAllPsopsHandler | QUERY_ALL | 查询所有psop |
| GetPsopHandler | QUERY | 通过psop的id查询对应的psop |
| DeletePsopHandler | DELETE | 删除psop |

## FAQ
### 自定义处理器未生效
#### 现象描述：注册了自定义处理器，但调用时仍使用默认处理器。

#### 可能原因：
- 1.注册操作在获取处理器之后执行
- 2.注册的接口类型与实际使用的不匹配
- 3.自定义处理器未正确继承BaseHandle

#### 解决办法：
- 1.确保在应用启动时、任何业务处理之前完成注册
- 2.检查注册和获取时使用的InterfaceType是否一致
- 3.确认自定义处理器正确继承了BaseHandle并实现了handle方法

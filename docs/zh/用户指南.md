# 编排中心用户指南

## 1. 特性介绍

编排中心是一个面向多智能体（Agent）协作的可视化编排平台，支持通过图形化工作流设计器定义 Agent 之间的调用关系与执行流程。后端基于 Python 框架解析流程并驱动 Agent 协同工作，帮助用户高效构建、管理和运行复杂的 Agent 协作流程。

### 1.1 应用场景

- **多 Agent 协作流程编排**：当业务需要多个智能体协同完成复杂任务时，可通过可视化方式定义调用关系与执行顺序。
- **工作流模板管理**：支持将常用的协作流程保存为模板，便于复用和分享。
- **智能工作流生成**：根据自然语言描述或 PDF 文档自动生成工作流规划，降低编排门槛。
- **实时流程监控**：支持流式执行工作流并实时推送运行进展，便于调试和监控。

### 1.2 能力范围

对外提供以下 REST API 接口：

- **SOP 编排接口**：支持基于自然语言 SOP 文本或 PDF/TXT/MD 文件生成 PSOP 工作流。
- **意图编排接口**：根据自然语言意图描述，自动规划并生成可执行的 PSOP 工作流。
- **自动编排执行接口**：根据任务描述自动检索或生成 PSOP 并执行，返回 SSE 流式执行进度。
- **指定 PSOP 执行接口**：根据 PSOP ID 执行已知工作流，返回 SSE 流式执行进度。
- **Agent 列表查询接口**：获取系统中所有可用 Agent 及其技能信息。
- **执行结果查询接口**：根据执行 ID 查询工作流执行结果。

### 1.3 亮点特征

- **可视化编排**：提供图形化工作流设计器，通过拖拽和连线即可完成 Agent 协作流程设计，无需编写代码。
- **多模式生成**：支持 PDF 导入、手动编排、自然语言生成三种工作流创建方式，适配不同用户习惯。
- **智能检索**：基于自然语言意图检索历史工作流，快速复用已有流程。
- **实时流式执行**：通过 SSE 技术实时推送执行进度，便于前端展示和问题定位。

## 2. 启动服务

### 2.1 前提条件

- **Node.js**：20.19 或更高版本
- **Python**：3.10 或更高版本（用于启动示例 Agent 服务）

### 2.2 启动服务

#### 2.2.1 **启动注册中心服务**

   UI 界面展示的所有 Agent 信息均从注册中心获取。具体操作请参见：[注册中心的用户指南](https://gitcode.com/OpenAN/registry-center/blob/main/docs/zh/%E7%94%A8%E6%88%B7%E6%8C%87%E5%8D%97.md)。

#### 2.2.2 **启动编排中心后端服务**

方式一：使用脚本安装服务（Linux）

编排中心提供 `install_service.sh` 脚本，支持将编排中心安装为 systemd 服务，便于服务管理和开机自启。

1. 配置部署参数（可选）

脚本默认配置文件位于 `etc/systemd/deploy.conf`，可根据需要修改以下参数：

| 参数名称 | 默认值 | 说明 |
| --- | --- | --- |
| INSTALL_DIR | /opt/orchestration-center | 服务安装目录 |
| PYTHON_PATH | {INSTALL_DIR}/venv/bin/python3 | Python 解释器路径 |
| SERVICE_NAME | orchestration-center | systemd 服务名称 |
| INSTALL_DEPS | true | 是否自动安装依赖 |

2. 创建虚拟环境

进入项目目录，创建并激活虚拟环境：
```bash
cd {安装目录}/orchestration-center
python3 -m venv venv
source venv/bin/activate
```

3. 安装服务

使用脚本安装服务（需 root 权限）：
```bash
cd {安装目录}/orchestration-center/bin
sudo ./install_service.sh install
```

脚本支持的命令：

| 命令 | 说明 |
| --- | --- |
| install | 安装 systemd 服务 |
| uninstall | 移除 systemd 服务 |
| status | 查看服务状态 |
| start | 启动服务 |
| stop | 停止服务 |
| restart | 重启服务 |
| enable | 设置开机自启 |
| disable | 禁用开机自启 |

可选参数：
- `--dir=PATH`：指定安装目录
- `--python=PATH`：指定 Python 解释器路径
- `--no-deps`：跳过依赖安装

示例：
```bash
# 指定安装目录和 Python 路径
sudo ./install_service.sh install --dir=/opt/my-orchestration --python=/usr/bin/python3

# 跳过依赖安装（已手动安装依赖时使用）
sudo ./install_service.sh install --no-deps
```

4. 启动服务

安装完成后，使用以下命令启动服务：
```bash
sudo ./install_service.sh start
```

或使用 systemctl 命令：
```bash
sudo systemctl start orchestration-center
```

5. 查看服务状态

```bash
sudo ./install_service.sh status
```

或：
```bash
sudo systemctl status orchestration-center
```

6. 查看启动日志

```bash
journalctl -u orchestration-center -f
```

如果可以看到 `Uvicorn running on http://127.0.0.1:60000` 则表示启动成功。

## 3. 使用

### 3.1 前提条件

- 注册中心服务已启动并正常运行
- 编排中心后端服务已启动并正常运行
- 编排中心前端服务已成功启动
- 浏览器可正常访问编排中心前端页面

### 3.2 背景信息

编排中心提供三种工作流创建方式：

- **PDF 导入**：解析 PDF 文档中的流程描述，自动生成工作流
- **手动编排**：通过拖拽 Agent 卡片到画布并连线，可视化定义流程
- **自然语言生成**：输入业务意图描述，后端自动生成对应的工作流

执行工作流时，系统通过 SSE（Server-Sent Events）技术实时推送执行进度，前端可实时展示每个 Agent 的请求和响应。

### 3.3 使用限制

- 当前版本仅支持解析 PDF 文档中标题为 "5. Interaction Flow" 的章节
- 工作流 ID 在保存时自动生成，不支持自定义
- 示例脚本启动的 Agent 仅供 Demo 演示使用，生产环境需对接真实 Agent 服务

### 3.4 操作步骤

1. **访问编排中心界面**

   打开浏览器访问 `http://localhost:3003`

2. **配置服务地址**

   单击界面右上角的齿轮状图标，将 IP 修改为编排中心实际的后端服务 IP，端口修改为编排中心实际监听的端口，单击保存。

3. **查看 Agent 库**

   左侧区域展示全部Agent列表，支持按Agent名称或功能关键词进行搜索筛选；单击任意Agent后，右侧区域将展示该Agent的详细信息。

4. **创建新工作流**

   - 单击左侧上方的 `+` 按钮
   - 选择创建方式：

     | 方式 | 操作说明 |
     | --- | --- |
     | PDF 导入 | 上传 PDF 文件，系统自动解析并生成 PSOP |
     | 手动编排 | 将下方 Agent 卡片拖拽到画布，通过连线定义执行顺序；单击连线可设置跳转条件 |
     | 自然语言生成 | 输入业务意图描述，后台自动编排生成 PSOP |

    **PDF导入：**

    单击"SolutionPackage 导入"选项，选择一份 PDF 格式的文档上传，等待上传完成；
    上传完成后，用户可以编辑后再进行保存。

    **说明：** 支持导入的 PDF 文件必须是 SolutionPackage 类型。SolutionPackage 是 TM Forum 定义的标准解决方案包文档格式，用于描述电信行业的解决方案和交互流程。可在 [TM Forum 官网](https://www.tmforum.org/search?contentType=Asset&pageIndex=1&searchTerm=an%20l4%20solution%20package) 查阅相关标准文档和示例。
    
   **手动编排：**
    - 步骤一 添加Agent卡片：
    从界面下方区域将所需的 Agent 卡片拖拽至画布空白区域。
    - 步骤二 配置Agent技能：
    单击画布上已添加的Agent卡片，在弹出的选项中选择需要使用的技能项。
    - 步骤三 添加更多Agent：
    重复步骤一和步骤二，根据业务需求将多个Agent拖拽至画布。
    - 步骤四 连接Agent卡片：
    将鼠标悬停在某个 Agent 卡片右侧的“小蓝点”上，此时鼠标指针会变为“十”字形状。按住鼠标左键不放，拖动至另一张 Agent 卡片上，松开鼠标即可完成连线。
    - 步骤五 配置分支条件：
    单击需要配置分支的连线，在弹出窗中的“分支配置”提示框内输入跳转条件并保存。这样，后续执行将根据不同的条件走不同的分支路径。
    - 步骤六 保存工作流：
    所有配置完成后，单击画布上方的“保存”按钮，即可完成工作流的创建或更新。
    
    **自然语言生成：**

    单击“自然语言编排”选项，在弹出的输入框中输入一段自然语言描述，单击“GENERATE”按钮
    生成后的PSOP工作流会展示在当前窗口，默认是“编辑”状态，可以直接保存，也可以修改后再进行保存。

5. **保存工作流**

   完成编排后，系统自动保存或手动触发保存操作，返回的 `workflow_id` 可用于后续检索和执行。

6. **查看已有工作流**

   在左侧面板展示所有的PSOP，并通过上方的搜索框按名称快速查找。单击任一PSOP后，右侧区域将自动显示其详细信息。

7. **执行工作流**
- 在上方输入框中输入用户意图；
- 单击右侧的“检索工作流”按钮；
- 系统检索到对应PSOP后，左侧区域将显示该PSOP；
- 中间区域自动展示该PSOP对应的工作流；
- 单击左侧PSOP右侧的“▶”按钮，页面右侧将实时显示工作流的执行过程。

## 4. FAQ

### 4.1 为什么启动后左侧 Agent 库为空？

**现象描述**：进入编排中心界面后，左侧 Agent 库没有显示任何 Agent。

**可能原因**：

- 注册中心服务未启动
- 编排中心后端服务未正确配置注册中心地址
- 示例 Agent 服务未启动（演示环境）

**解决办法**：

1. 确认注册中心服务已正常启动
2. 检查编排中心后端配置文件中的注册中心地址是否正确
3. 如需查看 Demo，执行 `python -m samples.start_agents_server` 启动示例 Agent
